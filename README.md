## 💡 Interview Tips

### Technical Interviews
1. **Understand the question** - Ask clarifying questions
2. **Think aloud** - Explain your thought process
3. **Start simple** - Begin with basic solution, then optimize
4. **Consider trade-offs** - Discuss pros and cons
5. **Use examples** - Reference real projects when possible

### System Design Interviews
1. **Gather requirements** - Functional and non-functional
2. **Estimate scale** - Users, requests, data volume
3. **High-level design** - Components and interactions
4. **Deep dive** - Focus on critical components
5. **Discuss trade-offs** - CAP theorem, consistency vs availability

### Behavioral Interviews
1. **STAR method** - Situation, Task, Action, Result
2. **Leadership examples** - Demonstrate principal-level thinking
3. **Problem-solving** - Show how you handle challenges
4. **Collaboration** - Working with teams and stakeholders
5. **Impact** - Quantify your contributions
